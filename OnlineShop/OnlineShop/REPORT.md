# Доповідь до захисту проєкту «OnlineShop»

## 1. Загальний опис проєкту

**OnlineShop** — навчальний вебдодаток інтернет-магазину електроніки, розроблений на платформі **ASP.NET Core MVC** з використанням **Entity Framework Core** та бази даних **SQLite**. Проєкт демонструє реалізацію основних патернів веброзробки: MVC-архітектуру, ORM для роботи з базою даних, автентифікацію на основі cookies, серверний рендеринг сторінок через Razor Views та сервісний шар для бізнес-логіки.

**Мова програмування:** C#  
**Фреймворк:** ASP.NET Core 8.0  
**ORM:** Entity Framework Core 8.0  
**База даних:** SQLite  
**Фронтенд:** HTML5, CSS3, JavaScript  

---

## 2. Архітектура проєкту

Проєкт побудований за класичним паттерном **MVC (Model-View-Controller)** із додатковим сервісним шаром:

```
OnlineShop/
├── Models/          — моделі даних (сутності БД)
├── ViewModels/      — моделі представлень 
├── Data/            — контекст БД та ініціалізатор
├── Controllers/     — контролери (обробка HTTP-запитів)
├── Services/        — сервіси (бізнес-логіка)
├── Views/           — Razor-шаблони сторінок
└── wwwroot/         — статичні файли (CSS, JS)
```

### Розділення відповідальностей

- **Models** — визначають структуру даних та зв'язки між таблицями (User, Product, Category, CartItem, Comment, RecentlyViewedProduct).
- **Controllers** — приймають HTTP-запити, викликають сервіси або працюють з DbContext, формують модель для View.
- **Services** — інкапсулюють бізнес-логіку (CartService, RecentlyViewedService), що забезпечує повторне використання та тестування.
- **Views** — відповідають лише за відображення даних, використовують Tag Helpers для генерації HTML.
- **ViewModels** — передають дані з контролера до View, уникаючи надлишкової інформації.

## 3. Структура бази даних

База містить 6 таблиць:

| Таблиця | Призначення |
|---------|-------------|
| Users | Користувачі (з полем IsAdmin для розрізнення ролей) |
| Products | Товари з описом, ціною, зображенням та категорією |
| Categories | Категорії товарів |
| CartItems | Позиції кошика (зв'язок User ↔ Product) |
| Comments | Коментарі до товарів (зв'язок User ↔ Product) |
| RecentlyViewedProducts | Історія переглядів товарів користувачем |

### Зв'язки

- Product → Category: багато-до-одного (FK: CategoryId)
- CartItem → User, Product: багато-до-одного
- Comment → User, Product: багато-до-одного
- RecentlyViewedProduct → User, Product: багато-до-одного

Використано Fluent API в `AppDbContext.OnModelCreating()` для налаштування каскадного видалення та унікального індексу на Email.

## 4. Реалізований функціонал

### 4.1. Каталог товарів
Головна сторінка виводить всі товари у вигляді карток з фото, назвою, коротким описом, ціною та кнопками «Детальніше» і «В кошик». Реалізовано фільтрацію за категоріями (через chip-кнопки) та пошук за назвою.

### 4.2. Сторінка товару
Детальна сторінка показує повний опис, фото, ціну, блок коментарів, схожі товари (з тієї ж категорії, до 4 шт.) та останні 5 переглянутих товарів авторизованого користувача.

### 4.3. Кошик
Авторизовані користувачі можуть додавати товари, змінювати кількість (кнопки +/−), видаляти окремі позиції та очищати весь кошик. При повторному додаванні того ж товару збільшується кількість.

### 4.4. Авторизація
Реалізовано Cookie-автентифікацію: реєстрація з валідацією полів, вхід з перевіркою хешу пароля (SHA-256), вихід. Паролі зберігаються у вигляді хешів. Використано Claims для зберігання Id, UserName, Email та ролі.

### 4.5. Коментарі
Авторизовані користувачі можуть залишати текстові коментарі до товарів. Коментарі відображаються з автором та датою.

### 4.6. Останні переглянуті товари
При відкритті сторінки товару, він додається в таблицю RecentlyViewedProducts. При повторному перегляді — оновлюється час. Зберігається максимум 10 записів на користувача, відображається 5 без поточного.

### 4.7. Схожі товари
На сторінці товару виводяться до 4 товарів з тієї ж категорії (окрім поточного).

### 4.8. Адмін-панель
Користувач з роллю Admin (атрибут `[Authorize(Roles = "Admin")]`) має доступ до CRUD-операцій над товарами та категоріями. Видалення категорії неможливе, якщо в ній є товари.

## 5. Ключові технічні рішення

### Автентифікація
Замість Identity використано легку Cookie-автентифікацію з ClaimsPrincipal. Це спрощує проєкт, зберігаючи функціональність.

### Сервісний шар
CartService та RecentlyViewedService зареєстровані через DI-контейнер як Scoped-сервіси, що забезпечує інверсію залежностей та можливість тестування.

### Seed Data
`DbInitializer` автоматично створює базу, 4 категорії, 11 тестових товарів та адміністратора (admin@shop.com / Admin123) при першому запуску.

### Адаптивний дизайн
CSS побудований на Custom Properties (CSS-змінних), Flexbox та Grid. На мобільних пристроях навігація перетворюється на бургер-меню, таблиця кошика стає вертикальною.

## 6. Інструкція із запуску

```bash
# 1. Переконатися, що встановлено .NET 8 SDK
dotnet --version

# 2. Перейти в каталог проєкту
cd OnlineShop

# 3. Відновити залежності
dotnet restore

# 4. Запустити проєкт
dotnet run

# 5. Відкрити у браузері
```

База SQLite створюється автоматично через `EnsureCreated()`. Тестові дані заповнюються при першому запуску.

**Дані адміністратора:**  
Email: `admin@shop.com`  
Пароль: `Admin123`

## 7. Висновки

Проєкт демонструє практичне застосування ASP.NET Core MVC для побудови інтернет-магазину з усіма основними функціями: каталог, кошик, авторизація, коментарі, адмін-панель. Код організовано за принципом розділення відповідальностей, використано Entity Framework Core для роботи з базою, реалізовано адаптивний інтерфейс. Проєкт готовий до запуску та тестування.
